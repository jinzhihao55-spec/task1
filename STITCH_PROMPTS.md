# Stitch 前端生成提示词
# ========================
# 使用方法：将以下提示词复制到 Stitch 中生成前端页面
# ========================

---
提示词 1：整体平台首页（包含导航 + 5个功能入口卡片）
---

请为一个「多功能智能助手平台」设计并生成一个完整的单页面前端（HTML + CSS + JavaScript，打包为单个HTML文件）。

## 设计要求

### 整体风格
- 现代科技感，深色主题（深蓝/深灰背景 + 亮色点缀）
- 使用渐变色和毛玻璃效果（glassmorphism）
- 字体使用系统无衬线字体，标题加粗
- 整体配色：主色 #6366f1（靛蓝），辅助 #06b6d4（青色），背景 #0f172a
- 圆角卡片设计，柔和阴影

### 页面结构
1. **顶部导航栏**：固定在顶部，包含平台Logo「AI助手」和5个功能导航链接
2. **Hero区域**：大标题「多功能智能助手平台」，副标题「AI驱动的智能办公工具集」，配有简洁的动画背景
3. **功能卡片区**：5个功能入口卡片，网格布局（3+2排列），每个卡片包含：
   - 图标（使用emoji或SVG图标）
   - 功能名称
   - 简短描述
   - 「开始使用」按钮
4. **底部**：简单的版权信息

### 5个功能卡片内容
1. 📄 **简历优化** - 输入简历片段，AI帮你优化表达、量化成果
2. ✍️ **文案生成** - 描述场景，一键生成多种风格的专业文案
3. 🌐 **翻译助手** - 中英互译 + 润色，让表达更地道
4. 📑 **PDF摘要** - 上传PDF文档，AI快速提取核心要点
5. 📊 **CSV预览** - 上传表格数据，智能分析并生成报告

### 交互要求
- 卡片hover有上浮效果和光晕
- 导航链接点击平滑滚动到对应区域
- Hero区域有微妙的渐变动画背景
- 页面加载时有淡入动画
- 所有按钮有hover/active状态反馈

### 技术要求
- 单个HTML文件，CSS和JS内联
- 使用CSS Grid/Flexbox布局
- 响应式设计（适配手机和桌面）
- 不使用任何外部框架（纯HTML/CSS/JS）
- 所有文本使用中文

提示词 2：简历优化功能页
---

请为「简历优化」功能设计一个完整的交互页面（单个HTML文件，CSS和JS内联）。

## 设计要求

### 整体风格
- 与主页一致的深色科技风格（主色 #6366f1，背景 #0f172a）
- 左右分栏布局（桌面端），上下布局（移动端）

### 页面结构
1. **顶部**：返回主页按钮 + 功能标题「简历优化」
2. **左侧 - 输入区**：
   - 大文本输入框（textarea），placeholder提示「请粘贴你的简历内容...」
   - 字数统计显示
   - 「快速分析」按钮（调用 /api/resume/analyze）
   - 「AI优化建议」按钮（调用 /api/resume/optimize）
3. **右侧 - 结果区**：
   - 快速分析结果卡片（评分、关键词匹配、完整性检查）
   - AI优化建议展示区（Markdown渲染）
   - 「复制结果」按钮
4. **加载状态**：请求中显示骨架屏或加载动画

### API调用
```javascript
// 快速分析
POST /api/resume/analyze
Content-Type: application/json
Body: { "text": "简历内容" }

// AI优化
POST /api/resume/optimize
Content-Type: application/json
Body: { "text": "简历内容" }
```

### 交互要求
- 输入框自动调整高度
- 结果区域有打字机效果（逐字显示）
- 分析结果用进度条展示各项评分
- 错误提示用红色toast显示

提示词 3：文案生成功能页
---

请为「文案生成」功能设计一个完整的交互页面（单个HTML文件，CSS和JS内联）。

## 设计要求

### 整体风格
- 深色科技风格（主色 #6366f1，背景 #0f172a）
- 居中布局，最大宽度800px

### 页面结构
1. **顶部**：返回主页按钮 + 功能标题「文案生成」
2. **输入区**：
   - 场景描述输入框（textarea）
   - 风格选择（下拉菜单）：简洁有力 / 场景代入 / 数据驱动 / 情感共鸣 / 自定义
   - 生成数量选择（1-5个方案）
   - 「生成文案」按钮
3. **结果区**：
   - 多个文案方案卡片，每个卡片有编号和风格标签
   - 每个方案有「复制」「收藏」「重新生成」按钮
   - 底部有「改写」功能：选择一个方案，输入目标风格进行改写
4. **加载状态**：生成中显示创意灵感动画

### API调用
```javascript
// 生成文案
POST /api/copywriting/generate
Content-Type: application/json
Body: { "scene": "场景描述", "style": "简洁有力", "count": 3 }

// 改写文案
POST /api/copywriting/rewrite
Content-Type: application/json
Body: { "text": "原始文案", "style": "目标风格" }
```

### 交互要求
- 文案卡片有淡入动画，依次出现
- 复制成功显示绿色toast
- 风格标签用不同颜色区分

提示词 4：翻译助手功能页
---

请为「翻译助手」功能设计一个完整的交互页面（单个HTML文件，CSS和JS内联）。

## 设计要求

### 整体风格
- 深色科技风格（主色 #6366f1，背景 #0f172a）
- 左右分栏布局（源语言 | 目标语言）

### 页面结构
1. **顶部**：返回主页按钮 + 功能标题「翻译助手」
2. **语言选择栏**：
   - 源语言选择（自动检测 / 中文 / 英文）
   - 交换语言按钮（↔）
   - 目标语言选择（中文 / 英文）
3. **翻译区**（左右分栏）：
   - 左侧：源文本输入框 + 字数统计
   - 右侧：翻译结果展示 + 复制按钮
   - 底部：「翻译」按钮 + 「润色」按钮
4. **润色结果区**：显示润色建议和修改对比

### API调用
```javascript
// 翻译
POST /api/translate/translate
Content-Type: application/json
Body: { "text": "待翻译文本", "target_lang": "en" }

// 润色
POST /api/translate/polish
Content-Type: application/json
Body: { "text": "待润色文本", "lang": "zh" }
```

### 交互要求
- 语言交换按钮有旋转动画
- 翻译结果有淡入效果
- 支持快捷键 Ctrl+Enter 提交翻译

提示词 5：PDF摘要功能页
---

请为「PDF摘要」功能设计一个完整的交互页面（单个HTML文件，CSS和JS内联）。

## 设计要求

### 整体风格
- 深色科技风格（主色 #6366f1，背景 #0f172a）
- 居中布局

### 页面结构
1. **顶部**：返回主页按钮 + 功能标题「PDF摘要」
2. **上传区**：
   - 拖拽上传区域（虚线边框，支持拖拽和点击）
   - 文件类型限制提示（仅支持PDF）
   - 文件大小限制提示（最大16MB）
   - 上传进度条
3. **结果区**：
   - 文件信息卡片（文件名、大小）
   - AI摘要展示（Markdown渲染）
   - 「复制摘要」「下载摘要」按钮
4. **备选方式**：提供文本输入框，可直接粘贴文档内容进行摘要

### API调用
```javascript
// 上传PDF
POST /api/pdf/upload
Content-Type: multipart/form-data
Body: FormData with file field

// 文本摘要
POST /api/pdf/summarize
Content-Type: application/json
Body: { "text": "文档文本" }
```

### 交互要求
- 拖拽文件到上传区时有高亮效果
- 上传中显示进度条和文件图标动画
- 摘要生成后有展开动画

提示词 6：CSV预览功能页
---

请为「CSV预览」功能设计一个完整的交互页面（单个HTML文件，CSS和JS内联）。

## 设计要求

### 整体风格
- 深色科技风格（主色 #6366f1，背景 #0f172a）
- 全宽布局

### 页面结构
1. **顶部**：返回主页按钮 + 功能标题「CSV预览」
2. **上传区**：
   - 拖拽上传区域（支持CSV/TSV）
   - 文件信息提示
3. **数据预览区**：
   - 数据表格（带斑马纹、固定表头）
   - 列信息统计（类型、非空数、唯一值）
   - 分页控件（每页10/20/50行）
4. **AI分析区**：
   - 分析报告展示（Markdown渲染）
   - 数据质量评分（环形进度条）
   - 「复制报告」按钮

### API调用
```javascript
// 上传CSV
POST /api/csv/upload
Content-Type: multipart/form-data
Body: FormData with file field

// 文本分析
POST /api/csv/analyze
Content-Type: application/json
Body: { "data": "CSV格式文本" }
```

### 交互要求
- 表格支持横向滚动
- 列头可点击排序
- 数据质量用颜色编码（绿色=好，黄色=警告，红色=问题）
